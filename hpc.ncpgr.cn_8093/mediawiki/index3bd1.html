存储警报

系统每天都会统计/public目录使用情况，超过磁盘最大容量的80%时，就会邮件通知磁盘使用较多的用户清理数据。

登录节点警报

登录节点原则上不允许执行计算任务，只能提交作业。登录节点控制阈值设置如下，程序每隔20s进行一次扫描：

  • 系统内存整体使用高于 80%，按照内存使用排序，kill 占用内存最高的进程；
  • 60s 内 CPU 整体负载持续高于 85% 并且在扫描的时刻的 CPU 整体使用率占用高于 90%，两个条件同时满足时按照进程 CPU 占用排序，kill CPU 占用最高的进程；

注：若同时有 N 个占用 CPU 或者相同的程序，默认按照姓名排序

计算节点警报

系统会对所有计算节点进行监控，监控程序每隔半小时对集群进行一次扫描： 当某个计算节点的内存使用超过95%时，会发送警报邮件给该节点内存资源占用最高的用户。用户收到邮件后，请及时处理作业，以免该节点因内存耗尽而宕机。
