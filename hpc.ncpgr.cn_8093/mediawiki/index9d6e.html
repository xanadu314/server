集群R基本使用

集群上安装了多个版本的R，服务器用户可以根据需要，通过module命令调用需要的R 

module load R/3.3.1-foss-2016b

在这个R环境内，我预装了一些R包，但可能不全面，服务器用户可以在自己的目录下安装R包。由于software下的路径一般普通用户无写入权限，故安装R包时候，R包无法安装进software账号下的相关目录，默认情况下R包会装在~/R/x86_64-pc-linux-gnu-library/3.3内。 

> install.packages("genetics")
Warning in install.packages("genetics") :
  'lib = "/public/home/software/opt/bio/software/R/3.3.1-foss-2016b/lib64/R/library"' is not writable
Would you like to use a personal library instead?  (y/n) y
Would you like to create a personal library
~/R/x86_64-pc-linux-gnu-library/3.3
to install packages into?  (y/n) y

或 

> source ("http://bioconductor.org/biocLite.R")
Bioconductor version 3.4 (BiocInstaller 1.24.0), ?biocLite for help
A new version of Bioconductor is available after installing the most recent
  version of R; see http://bioconductor.org/install
> biocLite ("AffyRNADegradation") 
BioC_mirror: https://bioconductor.org
Using Bioconductor 3.4 (BiocInstaller 1.24.0), R 3.3.1 (2016-06-21).
Installing package(s) ‘AffyRNADegradation’

此外也可以在~/.bashrc目录内写入R_LIBS变量来指定R包安装的位置，如 

export R_LIBS=~/R_libs

可以使用下面的R函数来查看，目前可用的R包路径 

>.libPaths()

Microsoft R Open

Microsoft R Open 是微软在开源R的基础上进行了性能增强的版本，主要是利用了MKL数据库加速了数学计算，跑GWAS的同学可能会有所受益，另外还有个checkpoint的功能，可以尝试一下。集群上可以通过module调用Microsoft R Open 

module load R/microsoft-r-open-MRO-3.5.1

Microsoft R Open 默认使用全部计算核心进行多线程计算，实际因为各种原因用不到很高的核心数，建议用setMKLthreads(4)设置4核即可。 

> setMKLthreads(4)
> getMKLthreads() 
[1] 4
>
